import module


import module1[, module2[,... moduleN]

When the interpreter encounters an import statement, it imports the module if the module is present in the search path. A search path is a list of directories that the interpreter searches before importing a module.

Import statement
We can import a module using import statement and access the definitions inside it using the dot operator. Here is an example.

import math
print("The value of pi is", math.pi)

Output:
The value of pi is 3.141592653589793

We can also rename module. This can save our typing time.
#renaming math module as m
import math as m
print("The value of pi is", m.pi)


from...import statement
We can import specific names from a module without importing the module as a whole. Here is an example.

from math import sqrt
print("Square root of 16 is", sqrt(16))

WARNING: In general, avoid using the from..import statement, use the import
statement instead. This is because your program will avoid name clashes and will be
more readable.


The from...import * Statement
It is also possible to import all names from a module into the current namespace. Here is an example.

from math import *
print("The value of pi is", pi)

WARNING:Importing everything with the asterisk (*) symbol is not a good programming practice. This can lead to duplicate definitions for an identifier. It also hampers the readability of our code.


Module Search Path
While importing a module, Python looks at several places. Interpreter first looks for a built-in module then (if not found) into a list of directories defined in sys.path. The search is in this order.

- The current directory.
- Python then searches each directory in the shell variable PYTHONPATH.
- The installation-dependent default directory. On UNIX, this default path is normally /usr/local/lib/python/
